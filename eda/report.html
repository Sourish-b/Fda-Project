ML Methods & Justification

🌲
Method A — Random Forest Regressor (Supervised Learning)

Configuration

Estimators100 trees
Random State42
Train Split80%
Test Split20%
ShuffleYes
ScalingNone required

Why Random Forest?

  • Handles non-linear relationships between weather/climate inputs and energy output naturally
  • Ensemble method — averages 100 trees, reducing overfitting and variance
  • No strict feature scaling required for tree-based models
  • Natively produces Feature Importance scores (used in analysis)
  • Robust to outliers in climate data
The trained model is persisted as model/saved/rf_model.pkl and loaded at server startup. Predictions are served in real-time via the POST /api/predict endpoint. The model is only trained once — it checks for the .pkl file before training to avoid unnecessary recomputation.
🔵
Method B — K-Means Clustering (Unsupervised Learning)

Configuration

Clusters (k)2
Random State42
n_init10
ScalingStandardScaler
k SelectionElbow Method
FeaturesSolar, Wind, Biomass, Hydro

Why K-Means?

  • Unsupervised — discovers natural groupings without pre-labelled categories
  • k=2 chosen via Elbow Curve analysis (tested k=1 to 10)
  • State-level energy profiles (mean across years) used as input vectors
  • Cluster assigned human-readable labels: Energy Hub (high output) vs Energy Consumer (low output) based on mean Total_Renewable per cluster
  • Results visualized on a Folium choropleth map
The cluster label for each state is stored in model/saved/cluster_labels.csv and used by the dashboard to colour the India map and display the "Energy Hub" / "Energy Consumer" badge. The clustering uses StandardScaler (saved as scaler.pkl) since K-Means is distance-based and requires normalized features.

Data Pipeline & Processing Steps

1
Data Loading

Two sources loaded: the integrated CSV (renewable_energy_dataset.csv) and individual state Excel/CSV files from the STATEWISE_CLIMATE_RENEWABLEENERGY_DATA/ folder. Shape, dtypes, and null counts printed for diagnostics.

data_loader.load_integrated() · data_loader.load_all_states()

2
Data Cleaning & Standardisation

Column names converted to snake_case. State name aliases resolved (e.g. "Orissa" → "Odisha", "Delhi" → "NCT of Delhi"). Energy columns unified under canonical names (wind, solar, biomass, small_hydro). Rows missing all four energy columns are dropped.

preprocess.clean_data(df)

3
Feature Engineering

Total_Renewable is computed as the row-wise sum of all four renewable energy types. A state-level mean profile is generated by grouping on state name and computing the numeric column means — this collapses the time-series into one representative row per state.

preprocess.engineer_features(df) → saves state_profiles.csv

4
Feature Scaling

StandardScaler applied to the 9 climate features before K-Means clustering. The fitted scaler is saved as scaler.pkl for consistent transformation during inference. Note: the Random Forest does not require scaling.

preprocess.scale_features(df, feature_cols) → saves scaler.pkl

5
Train / Test Split (for Regression)

80/20 split with shuffle=True and random_state=42 for reproducibility. The 9 climate features form matrix X; Total_Renewable is the target vector y. Splits are saved as CSVs to avoid re-splitting on re-runs.

split.create_splits(df) → saves X_train.csv, X_test.csv, y_train.csv, y_test.csv

6
K-Means Clustering

Elbow curve plotted for k=1 to 10. k=2 selected. Cluster with higher mean Total_Renewable is labelled "Energy Hub"; the other is "Energy Consumer". Results saved and visualised on a Folium India map.

clustering.run_clustering(df_scaled, df_states) → saves cluster_labels.csv, elbow_curve.png, india_map.html

7
Random Forest Training & Evaluation

100-tree Random Forest trained on the training split. RMSE and R² computed on the held-out test set. Feature importance bar chart saved. Model persisted as rf_model.pkl. Actual-vs-Predicted and Residual plots generated for diagnostics.

train_rf.train_and_save(df) → saves rf_model.pkl, feature_importance.png

8
Seasonal Analysis

Monthly renewable output heatmap across all states generated. Top 5 states by mean output plotted as time-series trend lines. Stacked monthly energy mix charts generated for Karnataka and Tamil Nadu.

seasonal_analysis.plot_monthly_heatmap(df) · plot_top_states_trend(df) · plot_energy_mix(df, state)

9
Flask API & Interactive Frontend

All saved artefacts loaded at server startup. REST API exposes /api/states, /api/state/<name>, /api/seasonal/<name>, /api/summary, and POST /api/predict. Frontend renders the dashboard, choropleth map iframe, Chart.js visualisations, and manual prediction form.

backend/app.py (Flask) · frontend/index.html + charts.js

5

Model Evaluation Metrics

RMSE
Root Mean Square Error
Lower is better. Measures average prediction error in MW units.
R²
Coefficient of Determination
Closer to 1.0 is better. Proportion of variance explained by the model.
Elbow
Inertia (K-Means)
Used to select optimal k. Diminishing return after k=2 confirmed.
The evaluation scripts also generate Actual vs Predicted scatter plot (with ideal diagonal reference line and R² annotation) and a two-panel Residual Diagnostic plot (Residuals vs Predicted + Residual Histogram) for assessing homoscedasticity and normality of errors.
